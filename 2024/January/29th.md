# 【わかったこと】
・DBMSが式を評価するときには、常にいずれか1行にだけ注目している。<br>
・CASE演算子:列の値や条件式を評価し、その結果に応じて値を自由に変換してくれる。<br>
・式の評価と同様に関数の呼び出しも各行ごとに繰り返し行われる。<br>
・関数の呼び出しの記述は呼び出し完了後に戻り値に化ける。<br>
```
⚫︎SELECT文の基本構文

[ORDER BY 並び替え列名]
```
