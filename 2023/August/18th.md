# 【わかったこと】
・定数:変化しない(または頻繁に変化しない)値を扱うときに使用する。<br>
例）手数料率、消費税率など<br>
・定数は大文字で始まる必要がある。また慣習として全て大文字で書くことが多い。<br>
例）`TAX = 1.08`,`TAX_RATE = 1.08`<br>
・定数への代入は一度きりとすること。<br>
→警告メッセージは表示されるが再代入はできてしまうため注意すること。<br>
・リテラル:数値の123や`'Hello World!'`など、Rubyのプログラムの中に直接記述できる値のこと。<br>
```
⚫︎・リテラルの例
・数値
123
・文字列
'Hello World!'
・配列
[1, 2, 3]
・ハッシュ(連想配列)
colors = {'red' => '#ff0000', 'green' => '00ff00', 'blue' => '0000ff'}
```
・オブジェクト指向プログラミング言語とは<br>
①コンピュータ・プログラミングの概念の一つ。<br>
②オブジェクト指向の概念や手法を取り入れたもの。<br>
③プログラムをデータとその振舞が結び付けられたオブジェクトの集まりとして構成する、などといった特徴がある。<br>
・オブジェクト:データと処理の集まりのこと。<br>
・オブジェクト指向:オブジェクト同士が相互に関係し合うことで、システムの振舞を捉える考え方。<br>
・クラス:オブジェクトの設計図。<br>
・インスタンス:クラス(設計図)から作成した実態。<br>
・メソッド:クラスの中に定義されているもの。複数の処理を１つにまとめて扱いやすくしたもの。クラスに所属する関数のようなもの。<br>
・変数の手前にマイナスを付けると数値の正と負を反転できる。<br>
```
例）
> x = 1
 => 1 
> -x
 => -1 
```
```
・数値同士の割り算は整数になる。
> 3 / 2
 => 1
・小数点以下の数値が必要な場合はどちらかに小数点を付ける。
> 3.0 / 2
 => 1.5 
> 3 / 2.0
 => 1.5 
> 3.0 / 2.0
 => 1.5 
```
```
・冪乗はアスタリスクを2つ使用して行う。
> 10 ** 3
 => 1000
・整数を少数付きの数値に変換するには下記のように記述する。(Integer型からFloat型に変換する。)
> x = 3
 => 3 
> x.to_f
 => 3.0
> x.to_f / 2
 => 1.5 
> x / 2
・数値オブジェクトがどのようなクラスなのかを調べる場合、下記のように記述する。
> 1.class
 => Integer 
> 1.1.class
 => Float 
```
・文字列は「シングルクォート」または「ダブルクォート」で囲む。<br>
→ただし文字列に特殊文字がある場合挙動が異なるためその都度使い分ける。<br>
```
・改行文字「\n」がる場合ダブルクォートで囲む。
> puts "Ruby\nPrograming"
Ruby
Programing
 => nil
・下記のように特殊文字をシングルクォートで囲むと文字列として表示されてしまう。
puts 'Ruby\nPrograming'
Ruby\nPrograming
 => nil
・式展開:文字列の中で式や変数を展開すること。
例）ダブルクォートで囲んだ場合→式典会が行われる
> first_name = 'Hanako'
 => "Hanako" 
> last_name = 'Yamada'
 => "Yamada" 
> puts "My name is #{first_name} #{last_name}."
My name is Hanako Yamada.
 => nil
例）シングルクォートで囲んだ場合→式典会は行われず文字列として出力される
> first_name = 'Hanako'
 => "Hanako" 
> last_name = 'Yamada'
 => "Yamada" 
> puts 'My name is #{first_name} #{last_name}.'
My name is #{first_name} #{last_name}.
 => nil

⚫︎シングルクォートとダブルクォートについて
・特殊文字や式展開を使用するとき
→ダブルクォート(")
・それ以外のとき
→シングルクォート
```
・文字列の連結<br>
```
> puts 'Taro' + 'Yamada'
TaroYamada
 => nil

> puts 'Taro' + ' ' +  'Yamada'
Taro Yamada
 => nil

> first_name = 'Hanako'
 => "Hanako" 
> last_name = 'Yamada'
> puts first_name + ' ' + last_name
Hanako Yamada
```
```
・upcaseメソッドを使用することで小文字を大文字にすることができる。
例）
> s = 'Hello World!'
 => "Hello World!" 
> puts s.upcase
HELLO WORLD!
 => nil 
> puts s
Hello World!
 => nil
・「!」を付けることで破壊的メソッドになり変数の内容を書き換える。
例）
> puts s.upcase!
HELLO WORLD!
 => nil 
> puts s
HELLO WORLD!
 => nil 
```
```
・空白文字はいくつ連続しても動作に違いはない。
> 10 + 2 - 1
 => 11 
> 10+2-1
 => 11 
> 10  +  2  -  1
 => 11 
```
```
・数値に文字列を連結させる場合下記のように型を変換する必要がある。
> 1 + '1'.to_i
 => 2 
> 1 + '1.1'.to_f
 => 2.1
・文字列に「+」を使用して数値を連結することはできない。
→下記のように数値を文字列に「.to_s」メソッドを使用して変換する必要がある。
> 'Number is　' + number.to_s
 => "Number is　1
```
・インクリメント:値に1を足す。<br>
・デクリメント:値から1を引く。<br>
→RubyにJavaScriptでいう「++」、「--」はない。下記のように記述することで対応する。<br>
```
> n = 1
 => 1 
> n += 1 # n = n + 1 (インクリメント)
 => 2 
> n -= 1 # n = n - 1 (デクリメント)
 => 1 
```
・真偽値:命題の真偽を示す値。<br>
・Rubyの真偽値のルール<br>
①falseまたはnilであれば偽(false)<br>
②それ以外は全て真(true)<br>
例）<br>
(1)trueそのもの<br>
true<br>
(2)全ての数値<br>
1,0,-1<br>
(3)全ての文字列<br>
'abc','true','false',''<br>
・論理演算子:「&&」や「||」のような論理演算子を使用すると、複数の条件を一つにまとめることができる。<br>
・論理積(and)<br>
a && b<br>
→aとbがともに真の場合に真<br>
・論理和(or)<br>
a || b<br>
→aかbの少なくても1つが真の場合に真<br>
・「!」を使用することで真偽値を反転することができる。<br>
```
例）
> t1 = true
 => true 
> f1 = false
 => false 
> !t1
 => false 
> !f1
 => true 
```
```
・「?」を使用するメソッド
→真偽値を返すメソッド
例）文字列オブジェクトが空かどうかを調べる
> name = 'Nakamura'
 => "Nakamura" 
> name.empty?
 => false 
> name = ''
 => "" 
> name.empty?
 => true
・特定の文字が含まれているか調べる
例）
> name = 'Nakamura'
 => "Nakamura" 
> name.include?('m')
 => true 
> name.include?('z')
 => false 
```
