# 【わかったこと】
・JavaScript(略称:js)とは<br>
①プログラミング言語(スクリプト言語)の1つ。<br>
②プログラミング言語「Java」とは異なるもの。<br>
③実行環境が主にウェブブラウザに実装されている。<br>
④ウェブブラウザで動作する、(ほぼ)唯一のプログラミング言語。<br>
⑤ウェブページ内で動的な機能を実現できる技術。<br>
⑥JavaScriptは、現代のWebサービス開発には欠かせない。<br>
⑦Node.jsのようなプラットフォームを使えば、サーバーサイドで動作する処理も書ける。<br>
・console.log()関数:ブラウザのコンソールに出力する。<br>
```
・構文
console.log(<<<出力したいメッセージを記述>>>);
```
・HTML側で下記のように記述することでJavaScriptファイルから読み込むことができる。<br>
`<script src="main.js"></script>`<br>
・JavaScriptファイル内で「`'use strict';`」を宣言する(最初に宣言する)と、コードがstrict（厳格）モードで実行されるようになる。strictモードでは、より的確なエラーチェックが行われる。これまでエラーにならなかったような曖昧な実装がエラー扱いになる。<br>
・JavaScriptのコメント<br>
```
・1行の時は「//」を先頭に付ける。
// コメントしたいテキスト

・複数行の時はコメントしたい箇所を「/* */」で囲う。
/*
コメントしたい行1
コメントしたい行2
*/
```
・JavaScriptでの四則演算と余り<br>
```
・足し算「+」:10 + 3
・引き算「-」:10 - 3
・掛け算「*」:10 * 3
・割り算「/」:10 / 3
・余り「%」:10 % 3
```
・変数:値に名前を付けて一時的に保持できる入れ物。数値だけでなく文字列等も保持できる。値の変更が可能。「let」を用いることで変数を宣言できる。(文字列を宣言する際は「シングルクォート(')」または「ダブルクォート(")」で囲む。)<br>
```
・複数の変数を宣言する場合、下記のどちらで宣言してもよい。
①1つずつ宣言する。
let i;
let j; 
let k;
②1度にまとめて宣言する。
let i, j, k;
```
・定数:変化しない値に名前を付けて一時的に保持できる入れ物。数値だけでなく文字列等も保持できる。値を変更することは不可。constを用いることで定数を宣言できる。<br>
・変数の宣言方法2種類(旧:1種類)<br>

| キーワード | 再代入 | 再宣言 | 備考 |
| --- | --- | --- | --- |
| let | ○ | × |  |
| const | × | × | 定数<br>(際代入できない変数) |
| var | ○ | ○ | 古い(ES5)書き方。<br>新規利用は推奨されない。 |

・変数(定数)の命名規則<br>
①Unicode文字、数字、記号（_と$）のみを組み合わせた名前にする<br>
②はじめの文字に数字は使用できない<br>
③予約語と同じ名前は変数名に使用できない<br>
④大文字と小文字は区別される<br>
→Javaでは慣例としてメソッド名や変数名を定義する際にローワーキャメルケースを用いる。<br>
・キャメルケース:アルファベットで複合語やフレーズを表記する際、各単語や要素語の先頭の文字を大文字で表記する手法のこと。<br>
例）favoriteFood<br>
・ローワーキャメルケース:キャメルケースのうち、最初の一文字を小文字から始める命名規則のこと。<br>
例）camelCase<br>
・エスケープシーケンス:通常の文字列では表せない特殊な文字や機能を表現するための方法。<br>
```
・下記ではエスケープシーケンスを使用することでシングルクォートを表示させている。
console.log('Hello I\'m Yuta Nakamura.');

・文字列を表示させる際にダブルクォートを用いることでシングルクォートを表示させることができる。
console.log("Hello I'm Yuta Nakamura.");

・文字列を表示させる際にシングルクォートを用いることでダブルクォートを表示させることができる。
console.log('"Nakamura"-san.');

・文字列を表示させる際にはシングルクォートまたはダブルクォートのどちらを使用してもよい。(特に決まりはない。迷った場合はシングルクォートの方がよい。)
→ただし同一プログラム内でどちらかに統一すること。
console.log('Hello');
console.log("Hello");
```
・インクリメント:数値の値を1増加させる。<br>
・デクリメント:数値の値を1減少させる。<br>
・条件分岐:値によって処理するプログラムを分岐させるもの。<br>
・条件分岐のパターン<br>
```
①if/else
・構文
if (条件式) {
  // 条件式がtrueの場合に実行したい処理
} else {
  // 条件式がfalseの場合に実行したい処理
}
・補足:条件式がfalseのときに実行する処置がない場合はelseを省略可能。
if (条件式) {
  // 条件式がtrueの場合に実行したい処理
}

②if/else if/else
・構文
if (条件式1) {
  // 条件式1がtrueの場合に実行したい処理
} else if (条件式2) {
  // 条件式2がtrueの場合に実行したい処理
} else {
  // 全ての条件式が成り立たなかった場合に実行したい処理
}
・補足:else ifは複数記述可能、
③switch:厳密等価演算子(===)による分岐に特化した条件分岐。つまり、対象となる1つの値を様々な値と比較して一致しているかどうか調べたいときに便利である。
・構文
switch (式) {
  case 値1:
    // 「式 === 値1」であるときに、実行したい処理を書く
    break; // ここでswitch文を抜けるという意味
  case 値2:
    // 「式 === 値2」であるときに、実行したい処理を書く
    break; // ここでswitch文を抜けるという意味
  default:
    // 式の値が全ての値に合致しないときに、実行したい処理を書く
    break;
}
・補足:caseは複数設置できる。defaultのbreakは省略可能。
```
・if,switchの使い分け:絶対的な1つの正解はない。<br>
→目安として<br>
①基本的にはifを利用。<br>
②厳密等価演算子(===)による評価のみかつ厳密等価演算子(===)による評価が3個以上の場合は、switchを使用するとスッキリと書ける場合が多い。<br>

| 演算子 | 説明 |
| --- | --- |
| == | 等しい |
| != | 等しくない |
| === | 厳密に等しい |

| > | より大きい |

・等価演算子(==)と厳密等価演算子(===)の違い<br>
```
・等価演算子(==):
数値と文字列を比較するとき、文字列は数値に変換される。JavaScriptは文字列の数値リテラル(プログラム上で数値や文字列など、データ型の値を直接記述できるように構文として定義されたもの)をNumber型の数値に変換しようと試みる。最初に、その文字列の数値リテラルから数学的な値を引き出す。次に、最も近いNumber型の値にこの値を丸める。つまり等価演算子(==)は文字列と数値の比較の場合、文字列を数値に変換してくれということである。

・厳密等価演算子(===):
オペランド(式で演算子の対象となっているもの)同士が、型を変換することなく厳密に等しいならば真を返す。

→実務では曖昧に比較したい理由を説明できる場合に限り等価演算子を使用する。通常は厳密等価演算子を使用すること。
```
